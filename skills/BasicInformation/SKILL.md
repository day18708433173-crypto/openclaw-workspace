# 基本信息录入助手 

收到录入基本信息指令时，提取字段 → 确认 → 写入智能表。

> 表配置（file_id/sheet_id/字段映射/格式速查）在 [TOOLS.md](../../TOOLS.md)，本 Skill 不重复。

---
## 流程

### Step 1：检查附件

- 若企微插件已提取文本（`finalContent` 含「文件内容预览」）→ 进入 Step 2
- 若只有「已收到文件：xxx.pdf」（二进制/扫描件）→ 回复：
  > 这个 PDF 是扫描件或图片格式，我暂时无法自动提取文字。请复制简历文本发送给我。
  >

### Step 2：LLM 提取

1.从简历文本中提取，输出 JSON：

```json
{
  "name": "候选人姓名",
  "phone": "电话",
  "email": "邮箱",
  "source_text": "简历中支持此判断的关键原文片段"
}
```
`channel` 只在简历中明确出现来源时才填对应枚举，否则填 "未知"。
2.从用户消息中提取岗位和渠道，输出 JSON：

```json
{
  "position": "应聘岗位",
  "channel": "渠道"
}
```

### Step 3：确认

向用户展示：

```
📋 从简历识别到：
- 姓名：张三
- 岗位：产品经理
- 电话：138xxxx
- 邮箱：zhang@example.com
- 渠道：BOSS
确认录入？回复"确认"写入，或直接修改后发送。
```

### Step 4：写入

用户确认后 → `smartsheet.add_records`（查重用姓名+电话组合键，同名同岗=重复不写入）。

回复：`✅ 已录入：张三 — 产品经理`

### Step 5：善后

删除临时 PDF 文件。

---

## 铁律

- **确认前不写入** — 用户未回复"确认"绝不调 `add_records`
- **提取不完整就追问** — 姓名或岗位缺一不可，追问缺失字段
- **查重后再写** — 先 `list_records` 判断是否重复
- **不猜值** — `channel` 不确定就是 "未知"，单选字段只写枚举内存在的值
