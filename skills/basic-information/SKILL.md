# 基本信息录入助手

收到录入基本信息指令时：拉取近 2 分钟 PDF → 逐份提取 → 校验 → 查重 → 写入智能表（可多人）。

> 表配置（file_id / sheet_id / 字段映射 / 格式速查）见 [TOOLS.md](../../TOOLS.md)，本 Skill 不重复。

## 背景（企微）

企微**无法对文件消息进行 @**。常见用法是：先发一份或多份简历 PDF，再另发「结合简历 / 录入基本信息 @机器人」。  
当前触发消息往往**没有附件**——必须从**同一会话近 2 分钟内**补拉 PDF；窗口内有几份就录入几人，禁止只因「本条没有文件」就放弃。

## 触发

`录入基本信息` `录入简历` `新增候选人` `结合简历`  
（简历在触发指令前的未 @ 文件消息里；可连续发多份再一次指令批量录入）

## 流程

### Step 1：从企微会话拉近 2 分钟内的简历 PDF

依赖 `wecom-msg`（`wecom_mcp`）。本会话首次调用前先按 `wecom-preflight` 检查白名单。

1. **定位当前会话**
   - 优先用当前企微会话元数据里的 `chatid` / 群 ID、以及单聊/群聊类型
   - 没有则 `get_msg_chat_list`（时间范围：今天），按会话名/上下文匹配；群聊 `chat_type=2`，单聊 `chat_type=1`
2. **拉近期消息**
   - `get_message`，时间范围严格为**最近 2 分钟**（`begin_time` = 当前时间 − 2 分钟，`end_time` = 当前时间）
   - 筛 `msgtype=file`，文件名以 `.pdf` 结尾（大小写不敏感）；也接受文件名含「简历」的 pdf
3. **选定文件（支持多人）**
   - 窗口内**全部**符合条件的 PDF 都纳入待录入列表，**不要只取最近一份**
   - 0 份 → 回复：近 2 分钟内没找到简历 PDF，请先发 PDF 再 @ 我说录入
   - ≥1 份 → 全部下载并进入后续步骤
4. **下载**
   - 对每份候选：`get_msg_media` 用 `file.media_id` 下载，拿到 `local_path`
   - 若无 `.pdf` 后缀，按 `content_type` / `name` 补全后重命名
   - 确认文件大小 > 0；单份失败则记下失败原因，继续处理其余份
   - **禁止**用 `MEDIA:` 把历史附件再发回用户
5. 对每份本地 PDF：调用 OpenClaw `pdf` 抽取文本  
   - prompt 示例：`提取简历全文，保留姓名、电话、邮箱等联系信息`  
   - 抽取过短/为空的那份标记为失败，继续处理其余份

> 用户已明确要求结合简历录入时，**直接下载解析全部候选**，不必再问「是否需要下载」。

### Step 2：LLM 提取（逐份 / 多人）

对 Step 1 成功抽到文本的**每一份**简历分别提取：

```json
{
  "name": "候选人姓名",
  "phone": "电话",
  "email": "邮箱",
  "source_text": "简历中支持此判断的关键原文片段",
  "source_file": "原 PDF 文件名"
}
```

岗位与渠道从**同一条用户触发消息**提取，**默认套用到本批所有候选人**：

```json
{
  "position": "应聘岗位",
  "channel": "渠道"
}
```

`channel` 仅在明确出现来源时填枚举值（BOSS / 猎聘 / 内推 / 校招），否则填 `"未知"`。

若用户在消息里按人名分别指定了不同岗位/渠道，则按人名匹配覆盖对应候选人；匹配不上的仍用默认值。

### Step 3：完整性校验（逐人）

五字段：`候选人姓名`、`电话`、`邮箱`、`应聘岗位`、`渠道`。

- 姓名、应聘岗位缺一不可；缺则该人**不写入**
- 电话/邮箱缺失 → 该人**不写入**，列入待补充
- 完整的人进入 Step 4；不完整的人汇总追问，**不要因为一个人缺字段就卡住整批已完整的人**

### Step 4：查重后写入（可批量）

1. `smartsheet.list_records` 查重：组合键 = **候选人姓名 + 应聘岗位**
2. 对每人：同名同岗 → 记为重复，跳过；未重复 → 加入待写入列表
3. 待写入列表一次性（或分批）`smartsheet.add_records`
4. 每条同步「最后更新时间」为当前毫秒时间戳；禁止写入「创建时间」
5. 汇总回复，例如：

```
✅ 已录入 2 人：张三 — 产品经理；李四 — 产品经理
⏭ 跳过重复 1 人：王五 — 产品经理
⚠️ 待补充 1 人：赵六（缺电话、邮箱）— 请补充后再说录入
```

仅 1 人且成功时仍可用简式：`✅ 已录入：张三 — 产品经理`

### Step 5：善后

删除本批下载的全部临时 PDF。

## 铁律

- **只拉近 2 分钟** — `get_message` 时间窗固定为最近 2 分钟，不扩大到小时/天
- **窗口内 PDF 全录** — 多份就多人，禁止只取一份后丢弃其余
- **完整的先写、缺的再问** — 不因部分人缺字段阻塞已完整候选人的写入
- **查重后再写** — 先 `list_records`；同名同岗跳过
- **不猜值** — `channel` 不确定就是 `"未知"`；单选只写枚举内存在的值（未知则不写渠道字段）
- **调用格式** — 字段映射与 mcporter 模板见 [TOOLS.md](../../TOOLS.md)
