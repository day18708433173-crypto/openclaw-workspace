# 招聘转化漏斗复盘

收到复盘指令 → 确定周期 → 拉数据 → 算指标 → 输出表格。

> 表配置（file_id/sheet_id/字段映射）见 [TOOLS.md](../../TOOLS.md)。

---

## 触发词

`招聘复盘` `月度复盘` `季度复盘` `半年复盘` `年度复盘` `转化漏斗` `招聘漏斗` `通过率` `复盘`

---

## 流程

### Step 1：确定周期

| 用户说 | 时间范围 |
|--------|---------|
| 月度复盘 / 本月 | 当前月 1 日 ~ 月末 |
| 季度复盘 / 本季度 | 当前季度首月 1 日 ~ 季末 |
| 半年复盘 / 上半年 | 1/1~6/30 或 7/1~12/31 |
| 年度复盘 / 今年 | 1/1 ~ 12/31 |

用户指定了具体时间（如"5月""Q1""去年"）则以用户为准。时间比较用毫秒时间戳。

如果用户没说周期，追问"要复盘哪个周期？月度/季度/半年度/年度？"

### Step 2：拉全量数据

**只做一件事：直接用 exec 执行下面这条命令。不要写临时文件，不要用 Node.js。**

```
mcporter call "tencent-saas-docs" "smartsheet.list_records" file_id:PTfdpsNTAQbnqbRxOP sheet_id:BB08J2 limit:100
```

> ⚠️ 参数用 `key:value` 格式，不要用 `--args` + JSON（Windows PowerShell 不兼容 JSON 单引号）。

如果 `has_more` 为 true，翻页拉完。拿到 JSON 后在内存中处理。

### Step 3：筛选 & 计算

按 `创建时间`（string_value，毫秒时间戳字符串）筛选周期内记录，然后统计：

```
初筛通过人数 = 周期内记录总数（录入即初筛通过）
一面通过人数 = 一面结果 = "通过" 的记录数
二面通过人数 = 二面结果 = "通过" 的记录数
OFFER接受人数 = OFFER是否接受 = "已接受" 的记录数

一面通过率   = 一面通过人数 / 初筛通过人数 × 100%
二面通过率   = 二面通过人数 / 一面通过人数 × 100%
OFFER接受率  = OFFER接受人数 / 二面通过人数 × 100%
总通过率     = OFFER接受人数 / 初筛通过人数 × 100%
```

分母为 0 时显示 "—"。百分比保留一位小数（如 `72.0%`）。

### Step 4：输出

用纯文本表格 + 一句话总结：

```
📊 招聘转化复盘 — 2026年7月

初筛通过：25 人
一面通过：18 人（72.0%）
二面通过：10 人（55.6%）
OFFER接受：6 人（60.0%）

总通过率：24.0%（6/25）

💡 一面→二面阶段流失最大（8人），建议关注面试难度或面试官匹配度。
```

---

## 铁律

- **只用 exec 跑一条命令** — 原样执行 Step 2 的命令，参数用 `key:value` 格式。不要 `--args` JSON、不要 `&&` 串联、不要 Node.js/Python 脚本
- **如果 exec 失败，不要写脚本绕过去** — 把错误信息发出来就行，我来修
- **全量拉取、内存过滤** — 拉完所有记录后在思考中计算
- **只读不写** — 不写智能表
- **分页拉完** — has_more=true 时继续拉
